# What Problem Did I Solve This Semester
<hr style="border: 1px blue; width: 50%;">

### How can we as software engineers use UI/UX to create a high quality product that fits the vision we have as developers while also making the product easy to use? 
  
### In other words, how do we manage complexity while not sacrificing our vision?

---

# What I Learned as a Result
<hr style="border: 1px blue; width: 50%;">

- To implement features that allow players to learn as they play, so they do not have to waste their time.
- UI toolkit in Unity for displaying these features and creating the theme for our video game.
- Better methods for user testing to get the feedback we need quickly and effectively.


---

# What I Discovered as a Result
<hr style="border: 1px blue; width: 50%;">

- This problem exists in many domains, but video games brings a unique challenge because the way infomation is displayed and told to players has to fit the needs of the game vision while still allowing for players to not feel lost when playing for the first time.
- This is different in other forms of software, where it is more common for all information to be shown in a way that highlights clarity above all else.
- Video games choose to obstruct information on purpose.
